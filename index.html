HTML

  • HTML Elements
  • The head element contains information (metadata) about the webpage.
  • The head element is not visible to user looking at webpage.
  • The title element defines the title for the webpage, is req'd and extremely important for SEO.
  • The style element defines the CSS styles for the website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata like character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.
  • The body element represents the visible content shown to the user.
  • The h1 to h6 elements represent larger more prominent titles/subtitles in a given text block.
  • The p element represents a paragraph or block of text. and will wrap most of the text on your pages
  • The ul ol and li elements represent unordered, ordered and list items.
  • The img element contains information about the image on a webpage. Most important is the src attribute which defines the location of the image.
  • The br element creates an empty line or line break.
  • The a element stands for anchor, and creates links to other webpages. The href attribute points to the URL.
  • !DOCTYPE html declaration tells browser what document to expect when opened. in this case html. not case sensitive.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Internal style sheet contains CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code.
  • External CSS style sheet is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • When using an external style sheet, we link the file using a link element placed in the HTML's head section.
  • Rather than repeating a rule combine two selectors into a single CSS rule with a comma, allowing the declarations to affect both selectors.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: commits all files in the working branch
  • git commit -m "add message here": associates a message with our commit; message would be replaced within quotes; short about changes implemented
  • git pull origin main: ensures current branch is up to date before opening a pull request
  • git push origin feature/add-html: pushes up all changes; replace feature/add-html with branch name
